1. Introduction

Wilfrid Laurier University uses an online learning management system called MyLearningSpace (MLS), which is a rebranded version of a product called BrightSpace, created by Desire2Learn (D2L). This educational platform allows students and instructors to communicate, share resources, and keep track of student progress. A vital part of this system is the feature that allows instructors to give feedback on student work. Currently the feedback system on MLS is very tedious and time consuming especially when creating individual feedback for large class sizes. The aim of this project is to make the feedback interface on MLS more user friendly and efficient.

1.1 Purpose

This document summarizes the goals and requirements of the ezGrader project. This includes the functions implemented, platforms used and etc. This document is created for developers as a basis for their work and for users looking to understand the functionality of this software.

1.2 Scope

ezMarker is a web platform that adds extended functionality to the MLS platform by setting student grades and feedback using the BrightSpace API. The new functionality will leave behind the conventional feedback uploading techniques such as manual entry or through a Dropbox interface. The user will start by selecting the course and grade item, then upload a formatted text file. This text file will be parsed by the ezMarker backend and upload the students’ grades and feedback through the Brightspace API. The goal of implementing a new system of feedback and grading is to add improve the efficiency of uploading student grades and feedback. With this software, professors and T.As/I.As will no longer have to enter marks and feedback one-by-one, making the process integrated and complete.

2. Overall Description

2.1 Product Perspective

ezMarker is a web-based application that allows users to set students’ grades in MLS through the use of the Brightspace API. Users will be able to set grades by uploading files from the Autograder or by manually inputting information into a webpage.

2.1.1 User Interfaces

  1. MLS Login Page
    • User will sign into MLS to provide authentication for use of the Brightspace API
  2. Course List Page
    • Shows all courses from current term that user has permission to set grades for
    • Selecting a course will hide/show the grade items
  3. Grade Item Page
    • User can upload an Autograder file (students to be graded will be determined by file)
    • User can also search for students and manually input grades and feedback through a web form
  4. Report Page
    • Displays outcome of set grades and feedback operation
    • Options to return to Course List Page or Logout

2.2 Product Functions

  1. Upload Autograder file to set grades and feedback in MLS for all students in a course
  2. Manually input grades and feedback to be set in MLS for selected students in a course

2.3 User Characteristics

Users will be MLS users with permissions to set grades for students (i.e. professors and TAs). No technical experience is required for the use of ezMarker.

2.4 Constraints

Development will be constrained by the capabilities of the Brightspace API.

2.5 Assumptions and Dependencies

ezMarker is dependent on continued support for Brightspace API and the Autograder outputting correct files.

3. Software Requirements

3.1 User Interfaces

3.1.1 The user signs into the website using their MLS login credentials

3.1.2 Once logged in, the user will be presented with a list of courses from the current term that the user has permission to set grades for. Courses should be sorted by course number.

3.1.3 Selecting a course will expand/collapse the course’s grade items

3.1.4 The Grade Item page will be opened once a grade item is selected.

3.1.4.1 A Choose File button will open file dialog for selecting an Autograder file.

3.1.4.2 An Upload Grades File button will upload the file and set the grades.

3.1.4.3 A search bar will appear below the above 2 buttons choosing students to manual input data for.

3.1.4.4 Selecting a student returned by the search bar will add Grade and Feedback input boxes (with the student’s name and ID acting as a label above them) to the page (underneath any existing students). An X icon next to the student’s ID will remove the input boxes from the page.

3.1.4.5 There will be a Select All button next to the search bar for quickly adding input boxes for all students.

3.1.4.6 A Set Grades button will use the manually inputted grades and feedback to set the grades.

3.1.5 A report page will show how many grades were successfully set. Any errors in the submission will also be displayed.

3.1.6 The report page has an Upload More Grades button, a Check Grades button (links to MLS page with all grades for that grade item), and a Logout button.

3.1.7 There will be a Logout button on the site’s banner.

3.1.8 The site needs to work on standard web browsers (Internet Explorer, Mozilla Firefox, Google Chrome, Safari, etc.)

3.2 Functional Requirements

3.2.1 Login. Occurs when users click the Login button on the Login page.

  • If the Brightspace API cannot identify the user, they will not be able to login and use the service. Similarly, if the user is identified but does not have permission to edit grades of any courses (determined by the Brightspace API), they will not be able to login and use the service.
  • Else, the user is brought to the Course List page

3.2.2 Upload/parse file. Occurs once Upload has been clicked and the user has selected a file in the file dialog.

  • If the file is not a correctly formatted Autograder file, an error message is displayed and the file is not parsed.
  • If a necessary parameter for a grade is missing, that grade will not be accepted.
  • Only grades for which the total grade is greater than or equal to the grade received will be accepted (i.e. 45/23 is not a valid grade). If the grade total is smaller than the grade received, the user will be prompted to continue (and skip the student) or modify the total of the grade item (taken to corresponding MLS page).
  • Grades will only be accepted for students that are currently enrolled in the selected course.
  • The results of the parsing will be relayed to the user on the Report page.

3.2.3 Search students. Occurs when user types into search bar on Grade Item page.

  • The search will be dynamic and will search students by name until a numeric character is entered, at which point student IDs are searched.

3.2.4 Get manually input data. Occurs when user has clicked Set Grades.

  • The data that has been input for the selected student(s) will be stored.
  • Only grades for which the total grade is greater than or equal to the grade received will be accepted (i.e. 45/23 is not a valid grade).
  • If no grade value has been entered for a student, the user will be prompted to choose if they want to go back and set a grade value or continue. If they continue, no grade will be set for the student.
  • Any unaccepted grades will be relayed to the user on the Report page.

3.2.5 Set grade and feedback data. Occurs once manually inputted data has been submitted or a formatted file has been uploaded.

  • Brightspace API will handle the backend for uploading the students' grades and feedback.

3.2.6 Logout. Occurs when user clicks Logout.

  • User’s session will be terminated and they will need to login again to access the service.

3.3 Performance Requirements

3.3.1 The site needs to be able to handle a file containing hundreds of records in a reasonable amount of time.

3.3.2 The website must be able to handle multiple concurrent users.

3.4 Security Requirements

3.4.1 Only those who have permission to set grades will be allowed past the login page.
