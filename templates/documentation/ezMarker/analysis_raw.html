1. Introduction

ezMarker is a web application that allows Wilfrid Laurier markers and professors to bulk upload grades and feedback from formatted text files into MyLearningSpace with the help of the Brightspace API supported by D2L. The purpose of ezMarker is to solve the issues and inefficiencies with manually uploading student grades and feedback through the current MyLearningSpace interface. On average, it can take markers a few hours to manually upload grades for one class. Marking time is cut down to minutes with the use of ezMarker's bulk file upload. This document outlines the structure of the classes and objects for the ezMarker application, along with the interactions between them.

2.2 Entity Objects

  • Host - Host information from config file.
  • Credentials - Stores user credentials.
  • User - Stores information about logged in user.
  • Course - Stores information about a course.
  • Student - Stores information about a student.
  • GradeItem - Stores information about a grade item. Abstract Class. Implimented by specific grade item type.
  • Grade - Stores information about a specific grade. Abstract Class. Implimented by specific grade type.

2.3 Boundary Objects

  • LoginView - Where user signs in with MLS credentials and receives Brightspace API token.
  • AvailableGradeItemsView - Where user selects a course and the grade item that they have permission to upload grades to.
  • UploadFileView - Where user can upload a file for bulk grade submission.
  • ManualGradingView - Where user can search for students and manually input grades and feedback.
  • ReportView - Where user will be shown how many grades were successfully set and if any errors occurred.
  • LogoutView - Where user signs out.

2.4 Control Objects

  • LoginController - Coordinates retrieval of Brightspace token and updating User object.
  • AvailableGradeItemsController - Coordinates the retrieval and displaying of Course and GradeItem objects the user has permission to upload grades to.
  • UploadFileValidationController - Coordinates form validation for uploading a file.
  • ParseUploadedFileController - Coordinates file parsing prior to grade upload.
  • ManualGradingValidationController - Coordinates form validation for manual grade input.
  • ParseManualGradingController - Coordinates manaul input parsing prior to grade upload.
  • UploadGradesController - Coordinates uploading grades through API and updating GradeItem object.
  • LogoutController - Coordinates signing out user and updating User object.
