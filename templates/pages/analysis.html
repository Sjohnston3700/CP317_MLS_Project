Introduction

ezMarker is a web application that allows Wilfrid Laurier markers and professors to bulk upload grades and feedback from formatted text files into MyLearningSpace, with the help of the Brightspace API supported by D2L. The purpose of ezMarker is to solve the issues and inefficiencies with manually uploading student grades and feedback through the current MyLearningSpace interface. On average, it can take markers a few hours to manually upload grades for one class. Marking time is cut down to minutes with the use of ezMarker's bulk file upload. This document outlines the structure of the classes and objects for the ezMarker application, along with the interactions between them.

Object Classification

Object Diagram

Entity Objects

  • Host - Stores host information from config file.
  • User - Stores information about logged in user.
  • Course - Stores information about a course.
  • Student - Stores information about a student.
  • GradeItem - Stores information about a grade item. Abstract Class. Implemented by specific grade item type.
  • Grade - Stores information about a specific grade. Abstract Class. Implemented by specific grade type.

Boundary Objects

  • LoginView - Where user signs in with MLS credentials and receives Brightspace API token.
  • LogoutView - Where user signs out.
  • CourseListView - Where user gets a list of courses that they have permission to upload grades to.
  • SearchStudentsView - Where user can search for students.
  • AvailableGradeItemsView - Where user selects a grade item of a course that they have permission to upload grades to.
  • ChangeGradeTotalView - Where user can change the grade total for a grade item.
  • IndividualGradingView - Where user can change the grade for an individual student's grade item.
  • BulkGradingView - Where user can either manually input grades or upload a file for bulk grade submission.
  • ReportView - Where user will be shown how many grades were successfully set and if any errors occurred.

Control Objects

  • LoginController - Coordinates retrieval of Brightspace token and updating User object.
  • LogoutController - Coordinates signing out user and updating User object.
  • CourseListController - Coordinates the retrieval of Courses that the user has permission to upload grades to.
  • SearchStudentsController - Coordinates searching of students by name or student ID.
  • AvailableGradeItemsController - Coordinates the retrieval of a course's GradeItems that the user has permission to upload grades to.
  • ChangeGradeTotalController - Coordinates changing of the grade total of a grade item and updating GradeItem object.
  • GradingController - Coordinates parsing, validating and uploading of students' grades and updating GradeItem object.
