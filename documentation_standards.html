Documentation Standards

Version 1.1

Last updated: December 18, 2017

Title

The title must have a large font size throughout the documentation.

Table of Contents

The table of contents must include subtitles that are listed under each main section Each section is to be numbered, and each subsection is the number of the section followed by a decimal point and the number of the subsection. For example, subsection 12 of section 1 will be numbered as 1.12. Each section and subsection is hyperlinked to the point of the page where the content begins

Introduction

The introduction must have a brief overview of the topic at hand. Any subsections should extend to the introduction rather than repeat or rephrase. The language should be simple to understand and break down complicated ideas. Any jargon, abbreviation, or uncommon phrases must be hyperlinked to a definition.

Terms and Definitions

This should include definitions and links to those definition if available. The extent of detail which the descriptions are written is so that a user is able to have a basic understanding of the term when used in a sentence.

References

The references should include the name of the referenced information, the date that the information is created, and where it was created. Any information that is not written by the group members and during the project timeline should be referenced.

Revision History

The history must include the dates that the website was revised, the name of the SQA and authors that contributed to that part of the revision, and actions that were done. Version 1 only specifies the authors and SQA. Subsequent versions simply contains bulleted (dashes) brief descriptions of the type of revision made.
