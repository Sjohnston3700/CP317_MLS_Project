1. Introduction

The Brightspace API Wrapper is a library used to make calls to the Brightspace API. It allows developers to fetch and push information through the API in a friendly manner. Rather than directly making HTTP requests, developers can simply use the functions available in the wrapper's library. HTTP requests consist of a URL and a verb. In this wrapper, requests GET or PUT data.

The wrapper is being developed primarily for ezMarker, a web application for uploading grades and feedback to a Brightspace environment. PHP and Python versions of the wrapper are being developed to support the corresponding versions of ezMarker.

1.1 Purpose

This document outlines the requirements of the Brightspace API Wrapper. It lists the API requests the wrapper contains. This document is intended for the use of current and future developers.

1.2 Scope

The Brightspace API is accessed using HTTP requests. For applications like ezMarker, which have many API calls, it is nice to have the HTTP requests hidden behind a wrapper. The wrapper takes Python/PHP arguments and returns Python/PHP objects, resulting in cleaner code because the application does not have to deal with complex HTTP requests and JSON objects.

The wrapper currently only includes the requests needed by ezMarker. The scope may later be increased, time-permitting, to include other key API requests, for the purpose of making the wrapper a useful resource for future development projects.

1.3 Definitions

Brightspace: A learning management system produced by D2L. It allows students and instructors to communicate, share resources, and keep track of student progress. A vital part of this system is the feature that allows instructors to set grades and give feedback on student work. The Brightspace API provides functionality for interacting with Brightspace environments.

D2L: A Waterloo-based company that produces learning management software. They are the creators of Brightspace.

3. Software Requirements

This section lists the classes and requests that the wrapper contains. Parameters and return objects have been specified in cases where the wrapper implementation of a request has significantly different input and output objects in comparison to the request.

3.1 Host

  • Host Contains API information
  • Uses Get API Versions (GET /d2l/api/versions/).

3.2 User

  • Contains user information
  • Stores the authentication token
  • Filters courses by given permissions
  • Uses Who Am I (GET /d2l/api/lp/(version)/users/whoami
  • Uses Get List of Accessible Courses (GET /d2l/api/lp/(version)/enrollments/myenrollments/)
  • Uses Get User Enrollments (GET /d2l/api/lp/(version)/enrollments/users/(userId)/orgUnits/
  • Uses Get Enrollments in OrgUnit (GET /d2l/api/lp/(version)/enrollments/myenrollments/(orgUnitId))

3.3 Course

  • Contains course information
  • Uses Get Course Enrollments (GET /d2l/api/lp/(version)/enrollments/orgUnits/(orgUnitId)/users/)
  • Uses Get Grade Items (GET/d2l/api/le/(version)/(orgUnitId)/grades/)

3.4 OrgMember

  • Contains information for a user's role in the BrightSpace organization such as ID, as well as grades and courses.

3.5 Grade Item

  • Contains Set Specific Grade Item (PUT /d2l/api/le/(version)/(orgUnitId)/grades/(gradeObjectId))

3.6 Grade

  • Contains Set Grade (PUT /d2l/api/le/(version)/(orgUnitId)/grades/(gradeObjectId)/values/(userId))
